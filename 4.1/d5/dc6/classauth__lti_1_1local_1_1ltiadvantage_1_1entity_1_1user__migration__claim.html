array $consumersecrets )

The migration_claim constructor.

The signature of a migration claim must be verifiable. To achieve this, the constructor takes a list of secrets corresponding to the 'oauth_consumer_key' provided in the 'https://purl.imsglobal.org/spec/lti/claim/lti1p1' claim. How these secrets are determined is not the responsibility of this class. The constructor assumes these correspond.

Parameters
array$jwtthe array of claim data, as received in a resource link launch JWT.
array$consumersecretsa list of consumer secrets for the consumerkey included in the migration claim.
Exceptions
coding_exceptionif the claim data is invalid.
